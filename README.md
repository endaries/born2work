# AI-асистент у Telegram — Етап 1 (кістяк)

Мінімальний, але робочий скелет: Telegram-бот на aiogram 3, який відповідає
через Claude API і має два шари пам'яті (коротка історія діалогу +
довгострокові факти).

## Структура

```
ai_assistant/
├── app/
│   ├── bot.py            # точка входу, хендлери повідомлень і команд
│   ├── claude_client.py  # виклик Anthropic API
│   ├── config.py         # читання .env
│   └── memory.py         # SQLite: короткострокова + довгострокова пам'ять
├── data/                  # тут з'явиться assistant.db після першого запуску
├── requirements.txt
└── .env.example
```

## Запуск

1. Встанови залежності:
   ```
   pip install -r requirements.txt
   ```

2. Скопіюй `.env.example` в `.env` і заповни:
   - `TELEGRAM_BOT_TOKEN` — створи бота через [@BotFather](https://t.me/BotFather),
     команда `/newbot`, скопіюй токен.
   - `ANTHROPIC_API_KEY` — з [console.anthropic.com](https://console.anthropic.com)
   - `ALLOWED_USER_IDS` — свій Telegram user_id (дізнатись можна через [@userinfobot](https://t.me/userinfobot)),
     щоб бот відповідав тільки тобі. Можна лишити порожнім на час тестів.

3. Запусти:
   ```
   python -m app.bot
   ```

4. Напиши боту в Telegram що завгодно — має відповісти.

## Команди бота

- `/start` — привітання і список команд
- `/remember <текст>` — явно зберегти факт у довгострокову пам'ять
  (наприклад: `/remember дедлайн по проєкту X — 30 липня`)
- `/facts` — показати всі збережені факти
- `/forget_all` — очистити коротку історію поточного чату (факти лишаються)

## Що навмисно просто на цьому етапі

- Факти зберігаються лише вручну через `/remember` — бот поки не вирішує
  сам, що варто запам'ятати. Автоматичне витягування фактів з розмови —
  наступний крок, і його краще робити усвідомлено (є ризик зберегти зайве
  або, навпаки, важливе пропустити).
- Пам'ять на SQLite, без векторного пошуку. Достатньо для десятків-сотень
  фактів; якщо стане більше — тоді є сенс переходити на embeddings.
- Одна модель (Claude), без tool use — бот поки тільки розмовляє,
  не виконує дій. Дії (пошук в інтернеті, календар, нагадування) —
  наступні етапи.
- Немає обробки голосових повідомлень, зображень, груп — це Етап 2.

## Наступні кроки (коли цей етап запрацює стабільно)

1. Голосові повідомлення: Whisper (транскрипція) на вхід, TTS на вихід
2. Робота в групових чатах (окрема логіка: коли бот повинен реагувати)
3. Нагадування/задачі: окрема таблиця + планувальник (APScheduler)
4. Веб-пошук (Anthropic web_search tool) як перший tool use
5. Gmail/Calendar через Google OAuth
